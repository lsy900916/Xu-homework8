行前实战营培训考核项目
XU-News-AI-RAG：个性化新闻智能知识库
一、功能要求
1.	实现定时任务机制，通过 RSS、网页抓取及智能代理工具合规获取新闻信息（严格遵守网络爬虫规范与安全准则），借助 n8n 定时触发节点与数据抓取节点完成自动化信息采集。
2.	基于 Ollama 本地部署大模型，推荐使用 qwen2.5:3b 模型，通过 n8n 的 HTTP 请求节点或自定义节点实现与 Ollama 服务的稳定交互。 
3.	构建本地 RAG 知识库系统，配置嵌入模型（推荐 all-MiniLM-L6-v2）、重排模型（推荐 ms-marco-MiniLM-L-6-v2）及大语言模型，利用 n8n 节点串联实现模型调用、数据处理与流程流转。
4.	通过 n8n 的 API 节点、数据库节点等，将抓取的新闻信息自动写入知识库系统， 支持 “ 结构化数据（Excel 可导入）” 与非结构化数据，实现数据自动化入库。 
5.	信息入库成功后，通过 n8n 邮件发送节点自动发送通知邮件（标题与内容可自定义），支持配置邮件模板与触发条件。
 
6.	提供用户登录功能，通过n8n 集成身份认证节点或对接外部认证服务实现。 
7.	登录后支持知识库内容管理：查看数据列表（可按类型 / 时间筛选）、单条 / 批量 删除、编辑元数据（标签、来源等），并支持页面上传多种类型数据；前端操作通过 API 与 n8n 工作流联动，实现数据管理自动化。 
8.	登录后提供语义检索功能：基于用户提问优先从本地知识库检索，通过 n8n 量数据库查询、相似度排序节点，按相似度返回结果。 
9.	若知识库无匹配内容，自动通过 n8n 触发联网查询（如调用百度搜索 API），获取前 3 条结果后，联动大模型推理节点处理并输出回答。
10.	登录后提供知识库聚类分析报告，展示关键词 Top10 分布；通过 n8n 数据统计、 聚类分析节点处理数据，生成结果并同步至前端展示。 



二、技术要求
本项目采用低代码 + 轻量化开发架构，完整技术栈如下，严格遵循 Agent Skill 框架
 
开发规范：
技术层级	选型方案	用途说明
前端框架	Vue / React（二选一）	可视化页面、数据展示、交互操作
后端框架	Flask	提供 API 接口、权限校验、 前后端数据中转 
关系型数据库	MySQL / SQLite（二选一）	存储用户信息、新闻元数据、 标签、来源等 
向量数据库	FAISS	存储文本向量，支持高效相似度检索
工作流工具	n8n（私有化部署）	全流程自动化编排、低代码节点联动
模型部署工具	Ollama	本地部署开源大模型，无公网传输
开发框架	Agent Skill	统一开发规范、智能代理能力封装
技术实现要点
1.	技术栈选型：前端 React/Vue，后端 Flask，关系库 MySQL/SQLite，向量库FAISS；核心流程由n8n 自动化编排，基于 Agent Skill 框架开发。 
2.  数据存储设计：元数据（ID、类型、来源、时间等）存入关系型数据库；向量数据 FAISS；n8n 通过数据库节点与自定义 FAISS 节点实现数据读写同步。 
3.	工作流集成：数据采集、入库、查询、分析、通知等核心逻辑全部用 n8n 作流实现，低代码完成异构系统联动。 
4.	大模型调用：通过 n8n HTTP 程内灵活调整。 
5.	身份认证：优先使用 JWT 保障安全可靠。 
6.	私有化部署与合规：n8n 私有化部署，所有数据本地处理、不上传第三方云，满足数据隐私合规要求。
